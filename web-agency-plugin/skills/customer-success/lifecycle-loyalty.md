# Lifecycle & Loyalty - Référence condensée

## Funnel Lifecycle

| Phase | Période | Objectif | Métrique clé | Cible |
|-------|---------|----------|-------------|-------|
| Onboarding | J0-J7 | Time-to-First-Value (TTFV) | Completion rate | >80% |
| Activation | J7-J30 | Aha moment atteint | Activation rate | >60% |
| Engagement | J30-J90 | Usage régulier | DAU/MAU (stickiness) | >20% |
| Retention | J90+ | Renouvellement | NRR >100%, GRR >90% | Churn <5%/an |
| Expansion | Continu | Upsell/cross-sell | Expansion rate | >20% ARR |
| Advocacy | Continu | Referrals, avis | K-factor, NPS promoteurs | K>1 |

## Séquence Onboarding J0-J7
- **J0** : Welcome email + setup guide (< 5 min première action)
- **J1** : Tip email première fonctionnalité clé
- **J3** : Check-in progrès, aide si bloqué
- **J5** : Cas d'usage avancé, social proof
- **J7** : Bilan onboarding, transition activation

## Activation - Aha Moment
Identifier par analyse de cohortes : action X corrélée à rétention J30 >80%.
Stratégies : progressive disclosure, milestone celebrations, parcours personnalisé.

## Engagement - Hook Model
`Trigger → Action → Récompense variable → Investissement`
- Engagement Score = Usage (40%) + Feature Adoption (30%) + Fréquence (30%)
- Re-engagement : J3 inactif → nudge, J7 → incentive, J14 → alerte CSM

## Retention J90+
- **Renewal timeline** : J-120 health check → J-90 prep → J-60 négo → J-30 closing → J-Day
- **Decision matrix** : Health score × Revenue → action (expand/maintain/save/let go)
- NRR = (MRR début + expansion - contraction - churn) / MRR début × 100

## Expansion - Signaux
| Signal | Score | Action |
|--------|-------|--------|
| Limite usage atteinte >80% | Fort | Proposition upgrade |
| Demande feature premium | Fort | Trial feature 14j |
| Croissance équipe client | Moyen | Proposition seats |
| Usage nouveau cas d'usage | Moyen | Cross-sell ciblé |

## Advocacy
- **Referral** : programme two-sided, reward progressif par tier
- **K-factor** = invitations × taux conversion (cible >1 pour viralité)
- **Avis** : demander post-moment positif (NPS 9-10, milestone, ticket résolu)
- **Case study** : 5 phases (sélection → interview → rédaction → validation → diffusion)

## Programmes Fidélité

### Structure Tiers (3-4 max)
| Tier | Distribution | Multiplicateur | Accès |
|------|-------------|----------------|-------|
| Base | 50-65% | x1 | Standard |
| Silver | 25-35% | x1.5 | Priority support |
| Gold | 10-15% | x2 | Exclusivités |
| Platinum | 1-5% | x3 | VIP, concierge |

### Mécanique Earn (acquisition points)
- **Transactionnel** (60-80%) : achats, abonnements, renouvellements
- **Engagement** (15-30%) : profil complété, avis, usage features
- **Social** (5-15%) : parrainage, partages, UGC
- Earn rate optimal : 1-3% du CA en valeur de points

### Mécanique Burn (récompenses)
Pyramide : Accessible 40% | Pratique 40% | Aspirationnel 15% | Exclusif 5%
- Ratio valeur perçue / coût réel > 2x
- Types : remises, produits, services, expérientiels, reconnaissance

### Gamification
- **Badges** : 6 catégories (commun→légendaire), rareté visible
- **Streaks** : récompense séries consécutives (grace period 1j)
- **Challenges** : daily (facile) → weekly → monthly → seasonal (ambitieux)
- **Leaderboards** : segments comparables, reset périodique

### Économie du Programme
- **Coût** = points émis × valeur unitaire + opérations + tech
- **ROI** = (lift fréquence + lift panier + lift rétention + réduction CAC) - coût
- **Liability IFRS 15** : provisionner points non utilisés
- **Break-even** : typiquement 12-18 mois
- **Anti-fraude** : cap earn/jour, délai burn, monitoring anomalies

### Checklist Lancement Programme
- [ ] Objectifs business définis (rétention +X%, fréquence +Y%)
- [ ] Tiers et seuils calculés sur données réelles
- [ ] Earn rate aligné sur marges
- [ ] Catalogue rewards testé (attractivité + rentabilité)
- [ ] Stack technique choisie (points engine, CRM, analytics)
- [ ] Plan communication lancement (teasing → launch → nurture)
