# Guides de Développement

Ce dossier contient les guides pratiques pour les développeurs. Ces documents définissent les standards et bonnes pratiques à suivre sur tous les projets.

## Guides Disponibles

| Guide | Description | Public |
|-------|-------------|--------|
| [onboarding-dev.md](./onboarding-dev.md) | Guide d'intégration pour nouveaux développeurs | Nouveaux arrivants |
| [definition-of-done.md](./definition-of-done.md) | Critères pour considérer une tâche terminée | Toute l'équipe |
| [code-conventions.md](./code-conventions.md) | Standards de code et conventions de nommage | Développeurs |

## Utilisation

### Pour un nouveau projet

1. Copiez les guides dans le dossier `docs/` de votre projet
2. Adaptez les sections avec les placeholders `___`
3. Ajoutez les informations spécifiques au projet
4. Référencez les guides dans le README principal

### Pour un nouveau développeur

1. Commencer par [onboarding-dev.md](./onboarding-dev.md)
2. Lire [code-conventions.md](./code-conventions.md) en détail
3. Garder [definition-of-done.md](./definition-of-done.md) comme référence

## Personnalisation

Ces guides sont des templates. Pour chaque projet :

- **Onboarding** : Remplissez les contacts, accès, et technologies spécifiques
- **DoD** : Ajustez les critères selon la maturité du projet
- **Conventions** : Adaptez selon le framework utilisé

## Contribution

Pour améliorer ces guides :

1. Discutez en rétrospective des points de friction
2. Proposez des modifications via PR
3. Validez avec l'équipe avant de merger

## Langues

Les guides sont en français. Pour une version anglaise :
- Créez un sous-dossier `en/`
- Traduisez les documents
- Maintenez la synchronisation des contenus
