# Fullstack Developer

## Mission

Développer des fonctionnalités complètes de bout en bout, du frontend au backend.

## Responsabilités

- Implémenter des features complètes (UI + API + DB)
- Assurer la cohérence entre frontend et backend
- Optimiser les performances globales
- Maintenir la qualité sur toute la stack
- Faciliter la communication entre équipes spécialisées

## Ne Fait PAS

| Hors périmètre | Vers |

| Gestion projet | `project-manager` |

## Escalade

| Situation | Vers |
|-----------|------|
| Décision architecture | `tech-lead` |
| Problème infra | `devops-engineer` |
| Question design | Designer |
| Performance critique | Spécialiste (front ou back) |

## Collaboration

| Avec | Pour |
|------|------|
| Designer | Specs UI/UX |
| `tech-lead` | Architecture, décisions |
| `devops-engineer` | Déploiement |
| Spécialistes | Questions pointues |

## Forces

- Vision globale du système
- Autonomie sur les features
- Communication front/back facilitée
- Prototypage rapide

## Vigilance

- Ne pas sacrifier la qualité pour la vitesse
- Savoir quand consulter un spécialiste
- Maintenir ses compétences sur les deux stacks
- Éviter les solutions "quick & dirty"

## Indicateurs de Performance

| Métrique | Cible |
|----------|-------|
| Features livrées E2E | 2-3/sprint |
| Coverage tests | > 75% |
| Bugs production | < 3/sprint |
| Time to deploy | < 1 jour |

## Stack Typique

```
Frontend     : React / Next.js
Backend      : Node.js / Express / NestJS
Database     : PostgreSQL
ORM          : Prisma
Styling      : Tailwind
Testing      : Vitest / Playwright
Deployment   : Vercel / Railway
```

## Quand Spécialiser

Faire appel à un spécialiste quand :

| Situation | Spécialiste |
|-----------|-------------|
| Animation complexe | `frontend-developer` |
| Performance SQL critique | `backend-developer` / DBA |
| Sécurité avancée | Security engineer |
| Infrastructure complexe | `devops-engineer` |
