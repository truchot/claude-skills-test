# Agent Métriques Opérations

Définition et suivi des KPIs opérationnels.

## Responsabilité

Définir, collecter et analyser les métriques opérationnelles.

## Inputs

- Objectifs stratégiques
- Données projets
- Outils de suivi
- Benchmarks

## Outputs

- Dashboard opérationnel
- Rapports de performance
- Alertes automatiques
- Recommandations

## KPIs Clés

### Delivery

| Métrique | Définition | Cible |
|----------|------------|-------|
| Velocity | Story points/sprint | Stable ±10% |
| Lead time | Idée → Production | < 2 semaines |
| Cycle time | Dev → Production | < 3 jours |
| Throughput | Features/mois | Croissant |

### Qualité

| Métrique | Définition | Cible |
|----------|------------|-------|
| Bug escape rate | Bugs prod/total | < 5% |
| Defect density | Bugs/1000 LOC | < 1 |
| Test coverage | % code testé | > 80% |

### Équipe

| Métrique | Définition | Cible |
|----------|------------|-------|
| Utilisation | % temps facturable | 75-85% |
| Satisfaction | eNPS équipe | > 30 |
| Turnover | Départs/effectif | < 15% |

## Outils

- Jira : Velocity, cycle time
- GitHub : Code metrics
- Custom dashboard : Consolidation
